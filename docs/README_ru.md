# ai-initializer

**Автоматический генератор контекста проекта для инструментов ИИ-кодирования**

> Сканирует директорию вашего проекта и автоматически генерирует
> `AGENTS.md` + контекст знаний/навыков/ролей, чтобы ИИ-агенты могли немедленно приступить к работе.

```
Одна команда → Анализ проекта → Генерация AGENTS.md → Работает с любым инструментом ИИ
```

---

## Использование

> **Предупреждение об использовании токенов** — При первоначальной настройке модель высшего уровня анализирует весь проект и генерирует несколько файлов (AGENTS.md, .ai-agents/context/, .ai-agents/skills/, .ai-agents/roles/). Это может потребовать десятки тысяч токенов в зависимости от размера проекта. Это единовременные затраты; последующие сессии загружают предварительно созданный контекст и запускаются мгновенно.

```bash
# 1. Попросите ИИ прочитать HOW_TO_AGENTS.md — он сделает всё остальное

# Вариант A: На английском (рекомендуется — меньше токенов, оптимальная производительность ИИ)
claude --dangerously-skip-permissions --model claude-opus-4-6 \
  "Read HOW_TO_AGENTS.md and generate AGENTS.md tailored to this project"

# Вариант B: На языке пользователя (рекомендуется, если вы планируете редактировать AGENTS.md вручную)
claude --dangerously-skip-permissions --model claude-opus-4-6 \
  "HOW_TO_AGENTS.md를 읽고 이 프로젝트에 맞게 AGENTS.md를 생성하라"

# Рекомендуется: --model claude-opus-4-6 (или новее) для наилучших результатов
# Рекомендуется: --dangerously-skip-permissions для непрерывного автономного выполнения

# 2. Начните работу со сгенерированными агентами
./ai-agency.sh
```

## Зачем вам это нужно?

### Проблема: ИИ теряет память в конце каждой сессии

```
  Сессия 1                  Сессия 2                  Сессия 3
┌───────────┐             ┌───────────┐             ┌───────────┐
│ ИИ читает │             │ ИИ читает │             │ Начинает  │
│ весь      │  Сессия     │ весь      │  Сессия     │ заново    │
│ код       │  завершена  │ код       │  завершена  │ с нуля    │
│ (30 мин)  │ ──────→     │ (30 мин)  │ ──────→     │ (30 мин)  │
│ Начинает  │ Память      │ Начинает  │ Память      │ Начинает  │
│ работу    │ потеряна!   │ работу    │ потеряна!   │ работу    │
│           │             │           │             │           │
└───────────┘             └───────────┘             └───────────┘
```

ИИ-агенты забывают всё по окончании сессии. Каждый раз они тратят время на понимание структуры проекта, анализ API и изучение соглашений.

| Проблема | Последствие |
|---|---|
| Не знает командных соглашений | Несоответствие стиля кода |
| Не знает полной карты API | Каждый раз исследует всю кодовую базу (затраты +20%) |
| Не знает запрещённых действий | Рискованные операции, например прямой доступ к продакшн БД |
| Не знает зависимостей между сервисами | Пропущенные побочные эффекты |

### Решение: Предварительно создать «мозг» для ИИ

```
 Начало сессии
┌──────────────────────────────────────────────────┐
│                                                  │
│  Читает AGENTS.md (автоматически)                │
│       │                                          │
│       ▼                                          │
│  «Я — эксперт по бэкенду этого сервиса»          │
│  «Соглашения: Conventional Commits, TypeScript   │
│   strict»                                        │
│  «Запрещено: изменять другие сервисы,            │
│   хардкодить секреты»                            │
│       │                                          │
│       ▼                                          │
│  Загружает файлы .ai-agents/context/ (5 секунд)  │
│  «Понято: 20 API, 15 сущностей, 8 событий»       │
│       │                                          │
│       ▼                                          │
│  Немедленно приступает к работе!                 │
│                                                  │
└──────────────────────────────────────────────────┘
```

**ai-initializer** решает эту проблему — сгенерируйте один раз, и любой инструмент ИИ мгновенно поймёт ваш проект.

## Основной принцип: трёхуровневая архитектура

```
                    Ваш проект
                         │
            ┌────────────┼────────────┐
            ▼            ▼            ▼

     ┌──────────┐  ┌──────────┐  ┌──────────┐
     │ AGENTS.md│  │.ai-agents│  │.ai-agents│
     │          │  │ /context/│  │ /skills/ │
     │ Личность │  │ Знания   │  │ Поведение│
     │ «Кто     │  │ «Что     │  │ «Как     │
     │  я?»     │  │  я       │  │  я       │
     │          │  │  знаю?»  │  │  работаю?│
     │ + Правила│  │          │  │          │
     │ + Права  │  │ + Домен  │  │ + Деплой │
     │ + Пути   │  │ + Модели │  │ + Ревью  │
     └──────────┘  └──────────┘  └──────────┘
      Точка входа  Хранилище памяти  Стандарты работы
```

### 1. AGENTS.md — «Кто я?»

**Файл идентичности** для агента, развёрнутого в каждой директории.

```
project/
├── AGENTS.md                  ← PM: Лидер, координирующий всё
├── apps/
│   └── api/
│       └── AGENTS.md          ← Эксперт по API: отвечает только за этот сервис
├── infra/
│   ├── AGENTS.md              ← SRE: управляет всей инфраструктурой
│   └── monitoring/
│       └── AGENTS.md          ← Специалист по мониторингу
└── configs/
    └── AGENTS.md              ← Менеджер конфигураций
```

Работает как **организационная структура команды**:
- PM контролирует всё и распределяет задачи
- Каждый участник команды глубоко разбирается только в своей области
- Они не занимаются напрямую работой других команд — они делают запросы

### 2. `.ai-agents/context/` — «Что я знаю?»

Папка, где **важные знания организованы заранее**, чтобы ИИ не приходилось читать код каждый раз.

```
.ai-agents/context/
├── domain-overview.md     ← «Этот сервис управляет заказами...»
├── data-model.md          ← «Существуют сущности Order, Payment, Delivery...»
├── api-spec.json          ← «POST /orders, GET /orders/{id}, ...»
└── event-spec.json        ← «Публикует событие order-created...»
```

**Аналогия:** Документация для адаптации нового сотрудника. Задокументируйте один раз — и никому больше не придётся это объяснять.

### 3. `.ai-agents/skills/` — «Как я работаю?»

**Стандартизированные рабочие инструкции** для повторяющихся задач.

```
.ai-agents/skills/
├── develop/SKILL.md       ← «Разработка фичи: Анализ → Проектирование → Реализация → Тест → PR»
├── deploy/SKILL.md        ← «Деплой: Тег → Запрос → Проверка»
└── review/SKILL.md        ← «Ревью: Безопасность, Производительность, Чеклист тестов»
```

**Аналогия:** Операционное руководство команды — заставляет ИИ следовать правилам, например «проверьте этот чеклист перед отправкой PR».

## Что писать, а что не писать

> ETH Zurich (2026.03): **Включение выводимого контента снижает успешность и увеличивает затраты на +20%**

```
         Пишите это                       Не пишите это
  ┌─────────────────────────┐     ┌─────────────────────────┐
  │                         │     │                         │
  │  «Используйте формат    │     │  «Исходный код находится│
  │   feat: для коммитов»   │     │   в папке src/»         │
  │  ИИ не может это вывести│     │  ИИ увидит это через ls │
  │                         │     │                         │
  │  «Нет прямого пуша      │     │  «React основан на      │
  │   в main»               │     │   компонентах»          │
  │  Правило команды,       │     │  Уже есть в официальной │
  │  не в коде              │     │  документации           │
  │                         │     │                         │
  │  «Требуется одобрение   │     │  «Этот файл содержит    │
  │   QA-команды перед      │     │   100 строк»            │
  │   деплоем»              │     │  ИИ может прочитать     │
  │  Процесс, не выводимый  │     │  это напрямую           │
  │                         │     │                         │
  └─────────────────────────┘     └─────────────────────────┘
       Пишите в AGENTS.md              НЕ пишите!
```

**Исключение:** «Вещи, которые можно вывести, но это слишком дорого делать каждый раз»

```
  например: Полный список API (нужно прочитать 20 файлов, чтобы понять)
  например: Связи модели данных (разбросаны по 10 файлам)
  например: Связи вызовов между сервисами (нужно проверить и код, и инфраструктуру)

  → Организуйте это заранее в .ai-agents/context/!
  → В AGENTS.md пишите только путь: «иди сюда, чтобы найти»
```

```
Включить (невыводимое)         .ai-agents/context/ (дорогой вывод)    Исключить (дешёвый вывод)
──────────────────────         ──────────────────────────────────      ─────────────────────────
Соглашения команды             Полная карта API                        Структура директорий
Запрещённые действия           Связи модели данных                     Содержимое отдельных файлов
Форматы PR/коммитов            Спецификации pub/sub событий            Документация фреймворков
Скрытые зависимости            Топология инфраструктуры                Связи импортов
```

## Как это работает

### Шаг 1: Сканирование и классификация проекта

Исследует директории до глубины 3 и автоматически классифицирует по шаблонам файлов.

```
deployment.yaml + service.yaml  →  k8s-workload
values.yaml (Helm)              →  infra-component
package.json + *.tsx            →  frontend
go.mod                          →  backend-go
Dockerfile + CI config          →  cicd
...19 типов определяется автоматически
```

### Шаг 2: Генерация контекста

Генерирует файлы знаний `.ai-agents/context/`, **фактически анализируя код** на основе обнаруженных типов.

```
Обнаружен бэкенд-сервис
  → Сканировать routes/controllers → Сгенерировать api-spec.json
  → Сканировать entities/schemas   → Сгенерировать data-model.md
  → Сканировать конфиг Kafka       → Сгенерировать event-spec.json
```

### Шаг 3: Генерация AGENTS.md

Генерирует AGENTS.md для каждой директории с использованием подходящих шаблонов.

```
Корневой AGENTS.md (Глобальные соглашения)
  → Коммиты: Conventional Commits
  → PR: Требуется шаблон, 1+ одобрений
  → Ветки: feature/{ticket}-{desc}
       │
       ▼ Автоматически наследуется (не повторяется в дочерних)
  apps/api/AGENTS.md
    → Переопределяет только: «Этот сервис использует Python»
```

Глобальные правила используют **паттерн наследования** — напишите в одном месте, и это автоматически применяется ниже.

```
Root AGENTS.md ──────────────────────────────────────────
│ Глобальные соглашения:
│  - Коммиты: Conventional Commits (feat:, fix:, chore:)
│  - PR: Требуется шаблон, минимум 1 ревьювер
│  - Ветка: feature/{ticket}-{desc}
│
│     Авто-наследование              Авто-наследование
│     ┌──────────────────┐       ┌──────────────────┐
│     ▼                  │       ▼                  │
│  apps/api/AGENTS.md    │    infra/AGENTS.md       │
│  (Указываются только   │    (Указываются только   │
│   дополнительные       │     дополнительные       │
│   правила)             │     правила)             │
│  «Этот сервис          │    «При изменении        │
│   использует Python»   │     значений Helm —      │
│                        │     сначала спроси»      │
└─────────────────────────┴──────────────────────────
```

**Преимущества:**
- Хотите изменить правила коммитов? → Измените только корневой файл
- Добавляете новый сервис? → Глобальные правила применяются автоматически
- Нужны другие правила для конкретного сервиса? → Переопределите в AGENTS.md этого сервиса

### Шаг 4: Загрузочные файлы для конкретных инструментов

Добавляет мосты к конфигурациям конкретных инструментов, чтобы **все ИИ-инструменты читали** сгенерированный AGENTS.md.

```
┌──────────────┐     ┌─────────────┐     ┌─────────────┐
│ Claude Code  │     │   Cursor    │     │   Codex     │
│  CLAUDE.md   │     │  .mdc rules │     │  AGENTS.md  │
│      ↓       │     │      ↓      │     │  (native)   │
│ "read        │     │ "read       │     │      ✓      │
│  AGENTS.md"  │     │  AGENTS.md" │     │             │
└──────┬───────┘     └──────┬──────┘     └─────────────┘
       └──────────┬─────────┘
                  ▼
           AGENTS.md (единый источник истины)
                  │
        ┌─────────┼─────────┐
        ▼         ▼         ▼
  .ai-agents/  .ai-agents/  .ai-agents/
   context/     skills/      roles/
```

> **Принцип:** Загрузочные файлы генерируются только для уже используемых инструментов. Файлы конфигурации для неиспользуемых инструментов никогда не создаются.

## Совместимость с инструментами

| Инструмент | Авто-чтение AGENTS.md | Bootstrap |
|---|---|---|
| **OpenAI Codex** | Да (нативно) | Не нужен |
| **Claude Code** | Частично (fallback) | Добавляет директиву в `CLAUDE.md` |
| **Cursor** | Нет | Добавляет `.mdc` в `.cursor/rules/` |
| **GitHub Copilot** | Нет | Генерирует `.github/copilot-instructions.md` |
| **Windsurf** | Нет | Добавляет директиву в `.windsurfrules` |
| **Aider** | Да | Добавляет чтение в `.aider.conf.yml` |

Автоматически сгенерировать загрузочные файлы:
```bash
bash scripts/sync-ai-rules.sh
```

```
project-root/
├── AGENTS.md                          # PM-агент (общая оркестрация)

│   │   ├── infra-spec.md              #   Helm-чарты, сеть, порядок деплоя
│   │   └── external-integration.md    #   Внешние API, аутентификация, лимиты

│   │   ├── review/SKILL.md            #   Ревью: на основе чеклиста
│   │   ├── hotfix/SKILL.md            #   Процесс экстренного исправления
│   │   └── context-update/SKILL.md    #   Процедура обновления файлов контекста
│   └── roles/                         # Определения ролей (глубина контекста для роли)

│       ├── backend.md                 #   Backend-разработчик

│       ├── sre.md                     #   SRE / Инфраструктура
│       └── reviewer.md               #   Ревьювер кода
│
├── apps/
│   ├── api/AGENTS.md                  # Агенты для каждого сервиса
│   └── web/AGENTS.md
└── infra/
    └── helm/AGENTS.md
```

```bash

=== AI Agent Sessions ===

  3) monitoring
  ...

Select agent (number): 2

=== AI Tool ===
  1) claude
  2) codex
  3) print

→ Agent automatically loads AGENTS.md and .ai-agents/context/
→ Ready to work immediately!
```

```bash
→ Switch panes with Ctrl+B N
```

**Пример api-spec.json:**
```json
{
  "service": "order-api",

    "method": "POST",

  }]

```

**Цель AGENTS.md:** Менее **300 токенов** после подстановки

## Протокол восстановления сессии

```
Начало сессии:
  1. Прочитать AGENTS.md (большинство инструментов ИИ делают это автоматически)
  2. Следовать путям к файлам контекста для загрузки .ai-agents/context/
  3. Проверить .ai-agents/context/current-work.md (незавершённая работа)
  4. git log --oneline -10 (понять последние изменения)

Конец сессии:
  1. Незавершённая работа → Записать в current-work.md
  2. Новые знания о домене → Обновить файлы контекста
  3. Незавершённые TODO → Записать явно
```

## Поддержание контекста

При изменении кода файлы `.ai-agents/context/` должны обновляться соответственно.

```
Добавлен/изменён/удалён API          →  Обновить api-spec.json
Изменена схема БД                    →  Обновить data-model.md
Изменена спецификация событий        →  Обновить event-spec.json
Изменена бизнес-политика             →  Обновить domain-overview.md
Изменена внешняя интеграция          →  Обновить external-integration.md
Изменена конфигурация инфраструктуры →  Обновить infra-spec.md
```

> Если не обновить, следующая сессия будет **работать с устаревшим контекстом**.

## Аналогия: Традиционная команда vs Команда ИИ-агентов

```
              Традиционная команда       Команда ИИ-агентов
              ────────────────────       ──────────────────────
 Лидер        PM (человек)               Root AGENTS.md (PM-агент)
 Участники    N разработчиков            AGENTS.md в каждой директории
 Онбординг    Confluence/Notion          .ai-agents/context/
 Инструкции   Вики команды               .ai-agents/skills/
 Роли         Должности/R&R-документы    .ai-agents/roles/
 Соглашения   Документы соглашений       Global Conventions (наследуются)
 Начало дня   Приход в офис              Сессия запущена → AGENTS.md загружен
 Конец дня    Уход (память сохранена)    Сессия завершена (память потеряна!)
 Следующий д. Память цела                .ai-agents/context/ загружен (память восстановлена)
```

**Ключевое отличие:** Люди сохраняют память после рабочего дня, а ИИ каждый раз забывает всё.
Именно поэтому существует `.ai-agents/context/` — это **долгосрочная память** ИИ.

## Результаты

| Файл | Аудитория | Назначение |
|---|---|---|
