# ai-initializer

**مولّد سياق المشروع التلقائي لأدوات البرمجة بالذكاء الاصطناعي**

> يفحص دليل مشروعك ويولّد تلقائياً
> `AGENTS.md` + سياق المعرفة/المهارات/الأدوار حتى تبدأ وكلاء الذكاء الاصطناعي العمل فوراً.

```
أمر واحد → تحليل المشروع → توليد AGENTS.md → يعمل مع أي أداة ذكاء اصطناعي
```

---

## الاستخدام

> **تنبيه حول استهلاك التوكنز** — أثناء الإعداد الأولي، يقوم نموذج متقدم بتحليل المشروع بالكامل وتوليد ملفات متعددة (AGENTS.md، .ai-agents/context/، .ai-agents/skills/، .ai-agents/roles/). قد يستهلك ذلك عشرات الآلاف من التوكنز اعتماداً على حجم المشروع. هذه تكلفة لمرة واحدة فقط؛ الجلسات اللاحقة تحمّل السياق المُعدّ مسبقاً وتبدأ فوراً.

```bash
# 1. اجعل الذكاء الاصطناعي يقرأ HOW_TO_AGENTS.md وسيتولى الباقي

# الخيار A: بالإنجليزية (موصى به — تكلفة توكنز أقل، أداء أمثل للذكاء الاصطناعي)
claude --dangerously-skip-permissions --model claude-opus-4-6 \
  "Read HOW_TO_AGENTS.md and generate AGENTS.md tailored to this project"

# الخيار B: بلغة المستخدم (موصى به إذا كنت تخطط لتحرير AGENTS.md يدوياً)
claude --dangerously-skip-permissions --model claude-opus-4-6 \
  "HOW_TO_AGENTS.md를 읽고 이 프로젝트에 맞게 AGENTS.md를 생성하라"

# موصى به: --model claude-opus-4-6 (أو أحدث) للحصول على أفضل النتائج
# موصى به: --dangerously-skip-permissions للتنفيذ المستقل دون انقطاع

# 2. ابدأ العمل مع الوكلاء المولَّدين
./ai-agency.sh
```

### المشكلة: الذكاء الاصطناعي يفقد ذاكرته في كل جلسة

```
 الجلسة 1               الجلسة 2               الجلسة 3
┌──────────┐         ┌──────────┐         ┌──────────┐
│ الذكاء   │         │ الذكاء   │         │ البدء    │
│ يقرأ     │  تنتهي  │ يقرأ     │  تنتهي  │ من       │
│ قاعدة    │ الجلسة  │ قاعدة    │ الجلسة  │ الصفر    │
│ الكود    │ ──────→ │ الكود    │ ──────→ │  مجدداً   │
│ (30 دقيقة│ فقدان   │ (30 دقيقة│ فقدان   │(30 دقيقة │
│ يبدأ     │الذاكرة! │ يبدأ     │الذاكرة! │ يبدأ     │
│ العمل    │         │ العمل    │         │ العمل    │
└──────────┘         └──────────┘         └──────────┘
```

تنسى وكلاء الذكاء الاصطناعي كل شيء عند انتهاء الجلسة. في كل مرة، تقضي وقتاً في فهم هيكل المشروع وتحليل واجهات API وتعلم الأعراف.

| المشكلة | العواقب |
|---|---|
| لا تعرف أعراف الفريق | تناقضات في أسلوب الكود |
| لا تعرف خريطة API الكاملة | تستكشف قاعدة الكود بالكامل في كل مرة (تكلفة +20%) |
| لا تعرف الإجراءات المحظورة | عمليات محفوفة بالمخاطر مثل الوصول المباشر لقاعدة بيانات الإنتاج |
| لا تعرف تبعيات الخدمات | آثار جانبية مغفولة |

### 1. AGENTS.md — "من أنا؟"

**ملف الهوية** للوكيل المنتشر في كل دليل.

```
project/
├── AGENTS.md                  ← PM: القائد الذي ينسق كل شيء
├── apps/
│   └── api/
│       └── AGENTS.md          ← خبير API: مسؤول عن هذه الخدمة فقط
├── infra/
│   ├── AGENTS.md              ← SRE: يدير كل البنية التحتية
│   └── monitoring/
│       └── AGENTS.md          ← متخصص المراقبة
└── configs/
    └── AGENTS.md              ← مدير الإعدادات
```

يعمل تماماً مثل **الهيكل التنظيمي للفريق**:
- يشرف PM على كل شيء ويوزع المهام
- كل عضو في الفريق يفهم منطقته بعمق فقط
- لا يتعاملون مباشرة مع عمل الفرق الأخرى — يطلبون ذلك

### 2. `.ai-agents/context/` — "ماذا أعرف؟"

مجلد تُنظَّم فيه **المعرفة الأساسية مسبقاً** حتى لا يضطر الذكاء الاصطناعي لقراءة الكود في كل مرة.

```
.ai-agents/context/
├── domain-overview.md     ← "هذه الخدمة تتعامل مع إدارة الطلبات..."
├── data-model.md          ← "توجد كيانات Order وPayment وDelivery..."
├── api-spec.json          ← "POST /orders، GET /orders/{id}، ..."
└── event-spec.json        ← "تنشر حدث order-created..."
```

**تشبيه:** وثائق التهيئة لموظف جديد. وثِّقها مرة واحدة، ولن يحتاج أحد لشرحها مجدداً.

### 3. `.ai-agents/skills/` — "كيف أعمل؟"

**أدلة سير عمل موحّدة** للمهام المتكررة.

```
.ai-agents/skills/
├── develop/SKILL.md       ← "تطوير المميزات: تحليل → تصميم → تنفيذ → اختبار → PR"
├── deploy/SKILL.md        ← "النشر: وسم → طلب → تحقق"
└── review/SKILL.md        ← "المراجعة: قائمة تحقق للأمان والأداء والاختبار"
```

**تشبيه:** دليل عمليات الفريق — يجعل الذكاء الاصطناعي يتبع قواعد مثل "تحقق من هذه القائمة قبل تقديم PR".

> ETH Zurich (2026.03): **تضمين المحتوى القابل للاستنتاج يقلل معدلات النجاح ويزيد التكلفة بنسبة +20%**

  مثال: علاقات الاستدعاء بين الخدمات (تتطلب فحص الكود + البنية التحتية)

  → نظِّم هذه مسبقاً في .ai-agents/context/!
  → في AGENTS.md، اكتب المسار فقط: "اذهب هنا للعثور عليه"

## كيف يعمل

### الخطوة 1: فحص وتصنيف المشروع

يستكشف الدلائل حتى العمق 3 ويصنّف تلقائياً بناءً على أنماط الملفات.

```
deployment.yaml + service.yaml  →  k8s-workload
values.yaml (Helm)              →  infra-component
package.json + *.tsx            →  frontend
go.mod                          →  backend-go
Dockerfile + CI config          →  cicd
...19 نوعاً يُكشف تلقائياً
```

### الخطوة 2: توليد السياق

يولّد ملفات المعرفة في `.ai-agents/context/` عبر **تحليل فعلي للكود** بناءً على الأنواع المكتشفة.

```
تم اكتشاف خدمة backend
  → فحص routes/controllers → توليد api-spec.json
  → فحص entities/schemas   → توليد data-model.md
  → فحص إعداد Kafka        → توليد event-spec.json
```

### الخطوة 3: توليد AGENTS.md

يولّد AGENTS.md لكل دليل باستخدام القوالب المناسبة.

```
Root AGENTS.md (الأعراف العالمية)
  → Commits: Conventional Commits
  → PR: قالب مطلوب، موافقة واحدة على الأقل
  → الفروع: feature/{ticket}-{desc}
       │
       ▼ موروث تلقائياً (لا يتكرر في العناصر الفرعية)
  apps/api/AGENTS.md
    → تجاوز فقط: "هذه الخدمة تستخدم Python"
```

تستخدم القواعد العالمية **نمط الوراثة** — اكتبها في مكان واحد، وتُطبَّق تلقائياً على التدرج.

```
Root AGENTS.md ──────────────────────────────────────────
│ الأعراف العالمية:
│  - Commits: Conventional Commits (feat:, fix:, chore:)
│  - PR: قالب مطلوب، مراجع واحد على الأقل
│  - الفرع: feature/{ticket}-{desc}
│
│     موروث تلقائياً           موروث تلقائياً
│     ┌──────────────────┐   ┌──────────────────┐
│     ▼                  │   ▼                  │
│  apps/api/AGENTS.md    │   infra/AGENTS.md    │
│  (قواعد إضافية فقط     │   (قواعد إضافية فقط  │
│   محددة)               │    محددة)            │
│  "هذه الخدمة تستخدم    │   "عند تغيير قيم Helm│
│   Python"              │    اسأل أولاً"        │
└────────────────────────┴───────────────────────
```

**الفوائد:**
- تريد تغيير قواعد commit؟ → عدِّل الجذر فقط
- تضيف خدمة جديدة؟ → تُطبَّق القواعد العالمية تلقائياً
- تحتاج قواعد مختلفة لخدمة معينة؟ → تجاوزها في AGENTS.md لتلك الخدمة

### الخطوة 4: Bootstrap خاص بالموردين

يضيف جسوراً إلى إعدادات خاصة بالموردين حتى **تقرأ جميع أدوات الذكاء الاصطناعي** AGENTS.md المولَّد.

```
┌──────────────┐     ┌─────────────┐     ┌─────────────┐
│ Claude Code  │     │   Cursor    │     │   Codex     │
│  CLAUDE.md   │     │  .mdc rules │     │  AGENTS.md  │
│      ↓       │     │      ↓      │     │  (native)   │
│ "اقرأ        │     │ "اقرأ       │     │      ✓      │
│  AGENTS.md"  │     │  AGENTS.md" │     │             │
└──────┬───────┘     └──────┬──────┘     └─────────────┘
       └──────────┬─────────┘
                  ▼
           AGENTS.md (مصدر الحقيقة الواحد)
                  │
        ┌─────────┼─────────┐
        ▼         ▼         ▼
  .ai-agents/  .ai-agents/  .ai-agents/
   context/     skills/      roles/
```

> **مبدأ:** تُولَّد ملفات Bootstrap فقط للموردين المستخدمين فعلاً. لا تُنشأ ملفات إعداد لأدوات غير مستخدمة.

→ Different agents work independently in each pane
→ Switch panes with Ctrl+B N
```

  1. العمل الجاري → سجّله في current-work.md
  2. المعرفة المكتسبة حديثاً بالنطاق → حدِّث ملفات السياق
  3. المهام غير المكتملة → سجّلها صراحةً
```

---

## صيانة السياق

عند تغيير الكود، يجب تحديث ملفات `.ai-agents/context/` وفقاً لذلك.

```
إضافة/تغيير/حذف API       →  حدِّث api-spec.json
تغيير مخطط DB             →  حدِّث data-model.md
تغيير مواصفات الأحداث     →  حدِّث event-spec.json
تغيير سياسة الأعمال       →  حدِّث domain-overview.md
تغيير التكامل الخارجي     →  حدِّث external-integration.md
تغيير إعداد البنية التحتية →  حدِّث infra-spec.md
```

> الفشل في التحديث يعني أن الجلسة التالية ستـ**عمل بسياق قديم**.

## تشبيه: الفريق التقليدي مقابل فريق وكلاء الذكاء الاصطناعي

```
              الفريق التقليدي           فريق وكلاء الذكاء الاصطناعي
              ────────────────────       ──────────────────────────────
 القائد       PM (إنسان)                Root AGENTS.md (وكيل PM)
 الأعضاء      N مطور                   AGENTS.md في كل دليل
 التهيئة      Confluence/Notion         .ai-agents/context/
 الأدلة       ويكي الفريق              .ai-agents/skills/
 تعريف الأدوار مسمى الوظيفة/وثائق R&R  .ai-agents/roles/
 قواعد الفريق وثائق أعراف الفريق       الأعراف العالمية (موروثة)
 بدء العمل    الوصول للمكتب            تبدأ الجلسة → يُحمَّل AGENTS.md
 نهاية العمل  المغادرة (الذاكرة محفوظة)  تنتهي الجلسة (الذاكرة تُفقد!)
 اليوم التالي الذاكرة سليمة            يُحمَّل .ai-agents/context/ (الذاكرة تُستعاد)
```

**الفرق الرئيسي:** يحتفظ البشر بذاكرتهم بعد مغادرة العمل، لكن الذكاء الاصطناعي ينسى كل شيء في كل مرة.
لهذا يوجد `.ai-agents/context/` — يعمل كـ**ذاكرة طويلة المدى** للذكاء الاصطناعي.
