# ai-initializer

**AI कोडिंग टूल्स के लिए स्वचालित प्रोजेक्ट संदर्भ जनरेटर**

> आपकी प्रोजेक्ट डायरेक्टरी को स्कैन करता है और स्वचालित रूप से
> `AGENTS.md` + ज्ञान/कौशल/भूमिका संदर्भ उत्पन्न करता है ताकि AI एजेंट तुरंत काम शुरू कर सकें।

```
एक कमांड → प्रोजेक्ट विश्लेषण → AGENTS.md निर्माण → किसी भी AI टूल के साथ काम करता है
```

---

## उपयोग

> **टोकन उपयोग सूचना** — प्रारंभिक सेटअप के दौरान, एक शीर्ष-स्तरीय मॉडल पूरे प्रोजेक्ट का विश्लेषण करता है और कई फाइलें उत्पन्न करता है (AGENTS.md, .ai-agents/context/, .ai-agents/skills/, .ai-agents/roles/)। प्रोजेक्ट के आकार के अनुसार इसमें हजारों टोकन खर्च हो सकते हैं। यह एक बार की लागत है; बाद के सत्र पहले से बने संदर्भ को लोड करते हैं और तुरंत शुरू होते हैं।

```bash
# 1. AI को HOW_TO_AGENTS.md पढ़ने दें और वह बाकी सब संभाल लेगा

# विकल्प A: अंग्रेजी (अनुशंसित — कम टोकन लागत, सर्वोत्तम AI प्रदर्शन)
claude --dangerously-skip-permissions --model claude-opus-4-6 \
  "Read HOW_TO_AGENTS.md and generate AGENTS.md tailored to this project"

# विकल्प B: उपयोगकर्ता की भाषा (अनुशंसित यदि आप AGENTS.md को मैन्युअल रूप से संपादित करने की योजना बनाते हैं)
claude --dangerously-skip-permissions --model claude-opus-4-6 \
  "HOW_TO_AGENTS.md를 읽고 이 프로젝트에 맞게 AGENTS.md를 생성하라"

# अनुशंसित: --model claude-opus-4-6 (या बाद का संस्करण) सर्वोत्तम परिणामों के लिए
# अनुशंसित: --dangerously-skip-permissions निर्बाध स्वायत्त निष्पादन के लिए

# 2. उत्पन्न एजेंट्स के साथ काम शुरू करें
./ai-agency.sh
```

## आपको इसकी आवश्यकता क्यों है?

AI कोडिंग टूल्स **हर सत्र में प्रोजेक्ट को शुरू से सीखते हैं**।

| समस्या | परिणाम |
|---|---|
| टीम की परंपराओं को नहीं जानता | कोड शैली में असंगतता |
| पूर्ण API मैप नहीं जानता | हर बार पूरा कोडबेस खोजता है (लागत +20%) |
| प्रतिबंधित कार्यों को नहीं जानता | जोखिमपूर्ण संचालन जैसे सीधे प्रोडक्शन DB तक पहुंच |
| सेवा निर्भरताओं को नहीं जानता | छूटे हुए साइड इफेक्ट्स |

**ai-initializer** इसे हल करता है — एक बार उत्पन्न करें, और कोई भी AI टूल आपके प्रोजेक्ट को तुरंत समझ लेता है।

## यह कैसे काम करता है

### चरण 1: प्रोजेक्ट स्कैन और वर्गीकरण

3 स्तर की गहराई तक डायरेक्टरी का अन्वेषण करता है और फाइल पैटर्न के आधार पर स्वचालित वर्गीकरण करता है।

```
deployment.yaml + service.yaml  →  k8s-workload
values.yaml (Helm)              →  infra-component
package.json + *.tsx            →  frontend
go.mod                          →  backend-go
Dockerfile + CI config          →  cicd
...19 प्रकार स्वचालित रूप से पहचाने जाते हैं
```

### चरण 2: संदर्भ निर्माण

पहचाने गए प्रकारों के आधार पर **वास्तव में कोड का विश्लेषण करके** `.ai-agents/context/` ज्ञान फाइलें उत्पन्न करता है।

```
बैकएंड सेवा पहचानी गई
  → रूट्स/कंट्रोलर्स स्कैन → api-spec.json उत्पन्न
  → एंटिटीज/स्कीमा स्कैन   → data-model.md उत्पन्न
  → Kafka कॉन्फिग स्कैन    → event-spec.json उत्पन्न
```

### चरण 3: AGENTS.md निर्माण

उपयुक्त टेम्पलेट्स का उपयोग करके प्रत्येक डायरेक्टरी के लिए AGENTS.md उत्पन्न करता है।

```
रूट AGENTS.md (वैश्विक परंपराएं)
  → कमिट्स: Conventional Commits
  → PR: टेम्पलेट आवश्यक, 1+ अनुमोदन
  → ब्रांच: feature/{ticket}-{desc}
       │
       ▼ स्वचालित रूप से विरासत में मिलता है (चाइल्ड में दोहराया नहीं जाता)
  apps/api/AGENTS.md
    → केवल ओवरराइड: "यह सेवा Python का उपयोग करती है"
```

### चरण 4: वेंडर-विशिष्ट बूटस्ट्रैप

वेंडर-विशिष्ट कॉन्फिग में ब्रिज जोड़ता है ताकि **सभी AI टूल्स** उत्पन्न AGENTS.md को पढ़ सकें।

```
┌──────────────┐     ┌─────────────┐     ┌─────────────┐
│ Claude Code  │     │   Cursor    │     │   Codex     │
│  CLAUDE.md   │     │  .mdc rules │     │  AGENTS.md  │
│      ↓       │     │      ↓      │     │  (नेटिव)    │
│ "read        │     │ "read       │     │      ✓      │
│  AGENTS.md"  │     │  AGENTS.md" │     │             │
└──────┬───────┘     └──────┬──────┘     └─────────────┘
       └──────────┬─────────┘
                  ▼
           AGENTS.md (एकल सत्य स्रोत)
                  │
        ┌─────────┼─────────┐
        ▼         ▼         ▼
  .ai-agents/  .ai-agents/  .ai-agents/
   context/     skills/      roles/
```

> **सिद्धांत:** बूटस्ट्रैप फाइलें केवल उन वेंडर्स के लिए उत्पन्न की जाती हैं जो पहले से उपयोग में हैं। अप्रयुक्त टूल्स के लिए कॉन्फिग फाइलें कभी नहीं बनाई जातीं।

```json
  "service": "order-api",
```

```
सत्र शुरू:
  1. AGENTS.md पढ़ें (अधिकांश AI टूल्स यह स्वचालित रूप से करते हैं)
  2. संदर्भ फाइल पथों का अनुसरण करके .ai-agents/context/ लोड करें
  3. .ai-agents/context/current-work.md जांचें (चल रहा काम)
  4. git log --oneline -10 (हाल के परिवर्तन समझें)

सत्र समाप्ति:
  1. चल रहा काम → current-work.md में दर्ज करें
  2. नया सीखा हुआ डोमेन ज्ञान → संदर्भ फाइलें अपडेट करें
  3. अपूर्ण TODOs → स्पष्ट रूप से दर्ज करें
```

## संदर्भ रखरखाव

जब कोड बदलता है, तो `.ai-agents/context/` फाइलों को तदनुसार अपडेट करना आवश्यक है।

```
API जोड़ा/बदला/हटाया        →  api-spec.json अपडेट करें
DB स्कीमा बदला              →  data-model.md अपडेट करें
इवेंट स्पेक बदला            →  event-spec.json अपडेट करें
व्यावसायिक नीति बदली        →  domain-overview.md अपडेट करें
बाहरी एकीकरण बदला          →  external-integration.md अपडेट करें
बुनियादी ढांचा कॉन्फिग बदला →  infra-spec.md अपडेट करें
```

> अपडेट न करने पर अगला सत्र **पुराने संदर्भ के साथ काम करेगा**।
